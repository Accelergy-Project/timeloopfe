\doxysubsection{Detailed Description}
Defines constraint macros to be used for simplifying constraint specification. 

Iteration constraint macros\+:
\begin{DoxyItemize}
\item factors\+\_\+only\+: Only the listed factors are allowed. Other factors are set to 1. e.\+g. factors\+\_\+only\+: \char`\"{}\+A=5\char`\"{} -\/$>$ factors\+: A=5, B=1, C=1, ...
\item no\+\_\+iteration\+\_\+over\+\_\+dataspaces\+: Takes a list of dataspaces. Iteration over all factors of these dataspaces is disabled. e.\+g. no\+\_\+iteration\+\_\+over\+\_\+dataspaces\+: \mbox{[}\char`\"{}\+Weights\char`\"{}\mbox{]} -\/$>$ factors\+: R=1 S=1 C=1 M=1 if the factors in weights are RSCM
\item must\+\_\+iterate\+\_\+over\+\_\+dataspaces\+: Takes a list of dataspaces. Must iterate over the union of factors of these dataspaces.
\item maximize\+\_\+dims\+: Takes a list of lists of dimensions. Each list of dimensions is maximized. e.\+g. maximize\+\_\+dims\+: \mbox{[} \mbox{[}R, S\mbox{]}, \mbox{[}C, M\mbox{]} \mbox{]} -\/$>$ find the largest factors of R and S that we can fit, then find the largest factors of C and M that we can fit. In a spatial constraint, this uses the fanout of the leaf. In a temporal constraint, this uses a maximize\+\_\+dims\+\_\+capacity directive.
\end{DoxyItemize}

Dataspace constraint macros\+:
\begin{DoxyItemize}
\item keep\+\_\+only\+: Only the listed dataspaces are kept. All other dataspaces are bypassed.
\item bypass\+\_\+only\+: Only the listed dataspaces are bypassed. All other dataspaces are kept. 
\end{DoxyItemize}
